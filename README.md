# InbarGil-Project

 הקוד מחולק למספר סעיפים, כאשר כל סעיף ממומש בקבצים נפרדים 
החלק הראשון של הפרויקט עוסק במחלקת Invoice. הקובץ Invoice.py מכיל את מחלקת Invoice, המייצגת חשבונית וכוללת נתונים כגון מספר חשבונית, שם לקוח, תאריך, מע"מ וסטטוס תשלום. המחלקה כוללת מתודות לחישוב הסכום הכולל ולהדפסת פרטי החשבונית. הקובץ MainInvoice.py משמש כקובץ הראשי להפעלת המחלקה, כולל קבלת קלט מהמשתמש והצגת הנתונים והחישובים.

החלק השני של הפרויקט עוסק בניירות ערך ומדגים שימוש בהורשה. הקובץ Security.py מכיל את מחלקת הבסיס Security ומספר מחלקות יורשות: Stock, Bond ו-Option. כל מחלקה יורשת תכונות בסיסיות ממחלקת הבסיס ומוסיפה התנהגות ייחודית בהתאם לסוג נייר הערך. הקובץ MainSecurities.py משמש להפעלת המחלקות, יצירת אובייקטים, חישוב ערכים והצגת התוצאות.

חלק נוסף בפרויקט עוסק בבדיקת כפולות. הקובץ Multiple.py כולל פונקציה הבודקת האם אחד משני מספרים הוא כפולה של השני, וכן תוכנית ראשית המדגימה עבודה עם קלט מהמשתמש, תנאים וטיפול בשגיאות.

בפרויקט נעשה שימוש בעקרון הכימוס (Encapsulation), שמטרתו להגן על נתוני האובייקט ולמנוע גישה ישירה אליהם. הגישה לנתונים מתבצעת באמצעות מתודות ו-properties של המחלקה בלבד, וכך נשמרת שלמות הנתונים ונמנעות טעויות. תוצאת הכימוס היא קוד מסודר, בטוח וקל לתחזוקה.

כמו כן, נעשה שימוש בעקרון ההורשה (Inheritance), שמטרתו לאפשר למחלקות חדשות להשתמש בתכונות ובפונקציות של מחלקת בסיס מבלי לשכפל קוד. בפרויקט זה מחלקות כמו Stock, Bond ו-Option יורשות ממחלקת Security. תוצאת ההורשה היא קוד ברור, קצר וקל להרחבה, עם התנהגות משותפת לצד מימוש ייחודי לכל מחלקה.
